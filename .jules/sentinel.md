## 2024-04-30 - Missing Navigation and Window Restrictions in Electron
**Vulnerability:** The Electron main window (`apps/desktop/electron/main.ts`) lacked navigation controls (`will-navigate`) and window creation restrictions (`setWindowOpenHandler`). If a renderer is compromised (e.g., via XSS) or an untrusted external link is clicked, it could allow an attacker to navigate the app's context to a malicious site or open unauthorized new windows.
**Learning:** In this Electron application, the `createWindow` function needs explicitly defined event listeners on `window.webContents` to ensure external URLs are routed through the system's default browser (`shell.openExternal`) rather than replacing the app's interface or spawning unauthorized new Electron windows.
**Prevention:** Always attach `setWindowOpenHandler` (returning `{ action: "deny" }`) and intercept `will-navigate` events on `webContents` to prevent arbitrary URL navigations and enforce strict window creation policies within Electron.
