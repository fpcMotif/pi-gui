## 2026-04-28 - Unrestricted Window Creation in Electron App\n**Vulnerability:** The Electron `BrowserWindow` implementation lacked a `setWindowOpenHandler` restrictor, which could potentially allow un-trusted web content to arbitrarily spawn new native Electron windows using standard web paradigms like `window.open` or `target="_blank"`.\n**Learning:** This is a common attack vector in Electron applications where missing navigation bounds allow arbitrary URIs (including local `file://` or executable shell commands via unsafe protocols) to bypass app constraints and execute via the underlying system.\n**Prevention:** Use `webContents.setWindowOpenHandler` inside every `BrowserWindow` instantiation to forcefully deny arbitrary window creation and restrict valid navigation requests explicitly to safe URL schemes (e.g. `http:` and `https:`) utilizing `shell.openExternal`.
