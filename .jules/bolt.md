## 2024-05-18 - Memoizing Expensive Markdown Rendering
**Learning:** In React, when a parent component manages state using mutable-like collections (e.g., `Set` for `expandedToolCallIds`), updating that state creates a new reference. This new reference causes the parent and all its children to re-render. If a child component performs an expensive operation like Markdown parsing (e.g., `react-markdown`), this re-render becomes a significant bottleneck, blocking the main thread even when the Markdown text itself hasn't changed.
**Action:** Always consider wrapping pure components that perform computationally expensive tasks (like Markdown or syntax highlighting) in `React.memo`. This is especially critical when they reside in large lists or frequently re-rendered parent components (like chat timelines).
